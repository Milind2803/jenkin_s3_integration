# 📘 CI/CD Documentation – Jenkins to AWS S3 Integration

This document outlines the high-level process for configuring a Jenkins pipeline to build an application and upload artifacts to an AWS S3 bucket.

---

## 🚀 Step 1: Create an S3 Bucket

Create a new Amazon S3 bucket that will be used to store build artifacts generated by the Jenkins pipeline.

---

## ⚙️ Step 2: Setup Jenkins Server

Provision and configure a Jenkins server with the required dependencies:
- Java
- Maven
- Jenkins

Ensure the Jenkins service is up and accessible.

---

## 🔐 Step 3: Access Jenkins

Log in to the Jenkins server using the web interface and complete the initial setup process.

---

## 🔌 Step 4: Install Required Plugins

Install the necessary Jenkins plugins to enable AWS integration and pipeline functionality:
- S3 Publisher
- Artifact Manager on S3
- Pipeline: AWS Steps

---

## 🔑 Step 5: Configure AWS Credentials

Add AWS credentials in Jenkins to allow secure access to AWS services.

---

## ⚙️ Step 6: Configure AWS Profile in Jenkins

Navigate to Jenkins settings, then go to **Manage Jenkins → System**, scroll to the bottom, and configure the AWS profile using the previously added AWS credentials.

---

## 📦 Step 7: Create Jenkins Pipeline

Create a new pipeline job in Jenkins to automate the build and deployment process.

---

## ⚡ Step 8: Generate S3 Upload Syntax

Use the Jenkins pipeline syntax generator to create the S3 upload step by specifying:
- File path
- S3 bucket name
- Destination path

---

## 🧾 Step 9: Configure Pipeline Script

Define the pipeline script to:
- Pull source code from the repository  
- Build the application using Maven  
- Upload the generated artifact to the S3 bucket  

### 📌 Example Pipeline

```groovy
pipeline {
    agent any

    stages {
        stage('Pull') {
            steps {
                git branch: 'main', url: 'https://github.com/Milind2803/EasyCRUD-Updated-k8s.git'
            }
        }

        stage('Build') {
            steps {
                sh '''
                cd backend
                mvn clean package -DskipTests
                '''
            }
        }

        stage('Deploy to S3') {
            steps {
                withAWS(credentials: 'aws-cred', region: 'ap-south-1') {
                    s3Upload(
                        bucket: 'mazhibucket123456789',
                        file: 'backend/target/student-registration-backend-0.0.1-SNAPSHOT.jar',
                        path: 'artifacts/app.jar'
                    )
                }
            }
        }
    }
}
```

---

## ▶️ Step 10: Build and Execute Pipeline

Trigger the pipeline and monitor execution through Jenkins. Verify successful completion and ensure that the artifact is uploaded to the configured S3 bucket.

---

## 📁 Step 11: Upload Documentation to GitHub

Add this documentation file to your repository and push it to GitHub for version control and collaboration.

---

## ✅ Outcome

- Jenkins pipeline successfully configured  
- Application artifact built and stored in S3  
- CI/CD workflow established  

---

## 🔒 Best Practices

- Use secure credential management  
- Avoid hardcoding sensitive information  
- Maintain proper naming conventions  
- Enable versioning and access control in S3
